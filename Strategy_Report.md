# “4%定投法（强化版）”自动化监控系统报告

## 一、 系统概述

本系统是基于“研究员雷牛牛”提出的“4%定投法（强化版）”逻辑开发的自动化投资辅助工具。它通过监控基金/股票的实时价格，严格执行“买跌不买涨”和“动态止盈”策略，旨在帮助投资者克服人性弱点，实现纪律化投资。

## 二、 核心功能模块

系统采用模块化设计，确保各功能组件的独立性与可维护性：

| 模块名称 | 功能描述 | 实现方式 |
| :--- | :--- | :--- |
| **数据抓取 (DataFetcher)** | 实时获取基金价格，支持多源容错。 | 集成 AkShare、新浪财经、天天基金 API。 |
| **策略引擎 (StrategyEngine)** | 执行 4% 定投法的核心逻辑计算。 | 严格匹配建仓、加仓及回撤止盈规则。 |
| **状态管理 (StateManager)** | 持久化记录持仓份额、平均成本及价格锚点。 | 使用 `state.json` 进行数据存储与同步。 |
| **消息推送 (Notifier)** | 在触发交易信号时发送即时通知。 | 支持飞书 Webhook（可扩展其他渠道）。 |
| **主调度 (Monitor)** | 协调各模块运行，完成巡检任务。 | Python 脚本统一驱动。 |

## 三、 策略执行逻辑详述

系统严格遵循以下三个阶段的逻辑判定：

### 1. 建仓期 (Initial Buy)
当 `invested_shares` 为 0 时，系统将当前价格记为“初始锚点”，并发出**首笔买入信号**。

### 2. 加仓期 (Buy on Dips)
* **触发条件**：当前价格 ≤ 上次买入价 × 96% (-4%)。
* **纪律性**：若未达到 -4% 阈值，即使市场波动，系统也**坚决不发出加仓信号**。
* **成本摊薄**：每次加仓后，系统会自动重新计算平均持有成本。

### 3. 止盈期 (Profit Taking)
* **观察区间**：当净值涨幅超过平均持有成本的 **15%** 时，进入“止盈观察期”。
* **动态止盈**：在观察期内，系统会持续追踪最高价。只有当价格从最高点**回撤达到 4%** 时，才会发出“全部卖出”的止盈信号。

## 四、 使用指南

### 1. 配置目标基金
在 `config.json` 中添加您想要监控的基金：
```json
{
  "target_funds": [
    {
      "name": "沪深300ETF",
      "code": "510300",
      "initial_investment_per_share": 1000,
      "max_shares": 10
    }
  ]
}
```

### 2. 设置环境变量
系统依赖飞书 Webhook 进行推送，请在运行环境中设置以下变量：
* `NOTIFIER_WEBHOOK`: 您的飞书机器人 Webhook 地址。

### 3. 运行监控
直接运行主脚本即可开始巡检：
```bash
python3 monitor.py
```

## 五、 总结与建议

本系统通过技术手段将“4%定投法”的复杂监控过程自动化，有效解决了“盯盘累”和“执行难”的问题。建议用户配合 GitHub Actions 使用，实现每日定时自动巡检，确保不漏掉任何一个关键的交易信号。
